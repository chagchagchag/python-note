매일 점심시간 마다 점심 안먹고 단백질 보충제 빨면서 정리하는 파이썬 노트 ~~~

ELK 를 잘 다루고 싶은 개인적인 욕심이 있고, 현재 듣고 있는 강의가 ELK 외에도 Spark, Airflow 하고 조화롭게 스트림 프로세싱을 하는 예제들도 많은데 결국에는 모든 코드들이 python 으로 이뤄진다. 

나 역시도 python 언어를 언젠가는 능수능란하게 다루고 싶었기에 공부할 때 점심 못 먹는다는 생각으로 억울한 적은 한번도 없었고 오히려 점심시간이 아까워서 집중했던 것 같다.

간단하게 디렉터리 내에 .pdf 파일들만 추려서 리스팅하는 프로그램을 짜야 하는 경우가 발생한다면... 
Java 는 클래스를 먼저 생성하고 각종 패키지를 import 해야 하고 클래스 레벨에서 분리해야 하기에 단순한 코드도 객체지향적으로 생각해야 한다는 불편함이 있는데 python 은 필요한 코드를 바로 작성해서 실천해볼 수 있다는 점이 장점으로 여겨졌다.

회사일로도 이것만 하면 얼마나 조을까 하고 혼자 상상해봄 ㅋㅋㅋ 장애 발생 로그들 착착 쌓아두고 검색 잘되게 하고 스트림 프로세싱 되게 하고 이런거 잘되면 너무 좋지

그리고 회사코드 보니까 사용자 인입 통계들도 죄다 관계형 Database 에서 논리적인 추론으로 죄다 Counting 하거나 서브쿼리, 인라인뷰로 도배되어 있던데, 이런 마케팅 관련 통계 데이터들도 ELK 스택 안에 안착 시키면 안정적으로 운용할 수 있을 것 같다. 내 목표는 ELK 하나만 잘하는게 목표가 아니다. 필요한 곳에 적절한 NoSQL 데이터베이스를 적용해볼수 있는 지식과 경험이 쌓였으면 하고 생각하고 있다.
