## 윈도우에서 파일 읽기 에러

목차 만들기 귀찮아서 디렉터리 내의 파일 명 리스트를 만드는 파이썬 프로그램을 만드는 중인데 윈도우에서는 파일 입출력을 파이썬으로 하기 쉽지 않다는 것을 느꼈다.

- [파이썬 권한 오류 / PermissionError : [Errno 13] Permission denied: ...](https://yjs-program.tistory.com/115)



<br/>



요약해보면 이렇다.

- 파일 경로가 이상하거나 파일이 아닌 폴더 경로를 지정할 때
- 파일의 확장자 명을 틀리거나 Windows, Ubuntu 상에서 경로의 디렉터리 구분자를 `\\`, `/` 를 서로 바꿔 사용하는 경우



<br/>



그냥 리눅스나 맥에서 써야 하는 언어구나 싶었다. 뭘 하려 해도 python 이 윈도우에서 잘 안된다. 윈도우 사용자의 열등감을 맥, 리눅스 사용자는 모른다 ㅋㅋㅋ<br/>

그래도 한국에서는 윈도우 써야 한다<br/>

그래서 결국은 이 문제를 해결했다.<br/>

<br/>



## 결론 (해결 방법)

일단 결론은 아래와 같이 해결했다.<br/>

파이썬이 설치된 경로로 이동한다.

```bash
$ cd ~/AppData/Local/Programs/Python/Python312
```

<br/>



그리고 권한을 777 로 수정해준다.

```bash
$ chmod 777 python.exe
```

## 이 외의 해결 가능한 방법들

아래와 같은 방법들이 있는데, 교과서적인 이야기이긴 하다. 블로그마다 다 아래 방법들이 적혀있길래 Open AI 들에게 물어본 결과가 도출된 대답들이 아닐까 싶다.

### 1) 접근하려는 파일/디렉터리 권한 문제인지 확인

그냥 누구든 이런말은 할 수 있다구.<br/>



### 2\) 권한 변경 python 코드 작성

```bash
import os
os.chmod('경로/파일명', 권한)
```

<br/>



e.g.

```python
import os
os.chmod('경로/파일명', 0o700)
```

<br/>



### 3\) 관리자 권한으로 실행

```bash
$ sudo python 프로그램명.py
```
